Worry-free deploying to GitHub Pages using Jekyll

Docs GitHub
Easy installation

Kickster is built for fast prototyping. Without the hassle of setting up your project from scratch and an easy deploy mechanism you can focus on the fun parts. Kickster uses the power of Jekyll which provides you with endless configuration possibilities.

Hassle-free deploying

Deploy in seconds from any GitHub branch to Github Pages with one simple command. JavaScript and CSS assets are minified and uglified using Jekyll Assets. Next to that CSS is autoprefixed for vendor browser support.

{% highlight shell %} # Installation ~ $ gem install kickster ~ $ kickster new site_name ~ $ cd site_name ~ $ bin/setup # Deployment ~ $ bin/deploy{% endhighlight %}
GitHub Pages hosting

Host your website for free from any of your GitHub repositories. Changes made to your GitHub Pages branch are live instantly. Perfect for blogging, project websites or documentation pages.

Automated testing

Enable Travis CI for your repository (optional) and push changes to your website safely. The included Travis CI config file tests if your jekyll build succeeds and checks if your HTML is valid with HTML Proofer.

Automated deploys

Kickster can be extended with automated deployment using Travis CI. By executing 3 simple steps your project will deploy automatically when changes are pushed to the master branch.

Use your custom plugins with GitHub Pages

Use any Jekyll plugin you wish, Kickster is perfect for users that require plugins that GitHub Pages does not support.
